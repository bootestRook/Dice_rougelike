## Context

当前主页由 `MainPrototypeView.gd` 动态生成，已有“开始游戏”和“退出”按钮。战斗侧已经存在 `Dice3DThrowLayer.gd`，可在真实战斗中播放 3D 物理投掷表现，但它依赖战斗数据、目标骰子槽位和规则层产出的 `RolledFace`。

本变更要补一个独立 GM 入口，用来快速观察最小物理投掷手感：一个方块从上方落下，受重力、碰撞和边界影响。它是正式骰子投掷动画的前置验证工具，而不是战斗规则的一部分。

## Goals / Non-Goals

**Goals:**

- 在主页提供中文“GM测试”入口。
- 提供一个 GM 功能列表，支持后续继续挂载调试功能。
- 提供“方块重力投掷”sandbox，用最小 3D 物理场景验证落下、翻滚、碰撞和重置。
- 保持 sandbox 与规则层、战斗流程、铸骰件和奖励系统解耦。
- 用 Debug 测试验证入口和 sandbox 基本行为。

**Non-Goals:**

- 不让方块投掷结果决定任何骰子点数、骰型、分数或奖励。
- 不在本变更中替换 `Dice3DThrowLayer.gd` 的战斗投骰实现。
- 不引入正式美术资源、外部物理库、第三方运行时或导出配置改动。
- 不新增血量、敌人攻击、锁定操作或任何战斗规则。

## Decisions

### Decision: GM 测试作为主页的独立视图

做法：在 `MainPrototypeView.gd` 中增加 `GM测试` 按钮，点击后切换到 `gm_test` 视图。该视图显示功能列表，并提供返回主页按钮。

理由：主页已经负责顶层导航，GM 入口从主页进入最直接；同时不会污染战斗界面和正式流程。

替代方案：在战斗选项菜单中加入 GM 入口。该方案能复用战斗层上下文，但会把调试工具和正式战斗 UI 混在一起，也不利于单独验证投掷前置功能。

### Decision: 方块投掷使用独立 sandbox 组件

做法：新增 `GMGravityThrowSandbox.gd`，内部创建 `SubViewportContainer`、`SubViewport`、`Camera3D`、灯光、`RigidBody3D` 方块、`StaticBody3D` 地面和边界。GM 功能列表只负责承载组件和按钮。

理由：sandbox 生命周期简单，目标是观察物理表现；独立组件可以被测试脚本直接实例化，也方便后续替换为骰子模型或复用其中的物理参数。

替代方案：直接复用 `Dice3DThrowLayer.gd`。该组件已经依赖骰子 view model、目标槽位和 face_index 校准，直接复用会让 GM sandbox 必须伪造战斗数据，反而降低调试价值。

### Decision: sandbox 只展示表现，不接触规则层

做法：方块投掷按钮只重置并释放一个刚体方块，记录调试状态，例如是否已投掷、是否创建刚体、是否发生落下或碰撞区域内停留。它不调用 `RollService`、`BattleController` 或 `RunState`。

理由：该工具用于观察“物理落下投掷”的前置手感。规则层仍由正式战斗流程负责，避免 UI 调试工具产生玩法状态。

替代方案：从 GM sandbox 直接生成随机骰面结果。该方案看起来更接近正式投骰，但会模糊 UI 表现和规则权威边界，因此不采用。

### Decision: UI 文案全部中文，内部节点名可英文

做法：按钮、标题、状态文本使用中文，如“GM测试”“方块重力投掷”“重新投掷”“返回主页”。节点名和脚本 class_name 可继续使用英文 ID。

理由：符合项目玩家可见文本必须中文的口径，同时保留内部代码可读性。

## Risks / Trade-offs

- [Risk] 主页按钮栏空间不足，新增按钮可能挤压现有按钮 → Mitigation: 实现时调整按钮尺寸或按钮栏宽度，并补主页 Debug 测试。
- [Risk] Headless 环境下无法判断真实渲染效果 → Mitigation: Debug 测试验证节点结构、刚体状态和投掷状态；人工视觉检查留到实施后的可运行窗口验证。
- [Risk] sandbox 与正式 `Dice3DThrowLayer` 参数漂移 → Mitigation: 先把 sandbox 定位为试验台，后续若参数稳定，再考虑抽取共享物理配置。
- [Risk] GM 入口误被视为正式玩家功能 → Mitigation: 标题和按钮文案明确使用“GM测试”，并保持功能列表与正式冒险流程分离。

## Migration Plan

1. 新增 GM 测试视图和方块重力投掷 sandbox。
2. 从主页接入 `GM测试` 按钮，并保留原有“开始游戏”“退出”行为。
3. 补充 Debug 测试并运行主页启动、GM sandbox 和主场景启动检查。
4. 如发现 GM 入口影响正式主页布局，可回退主页按钮接入，sandbox 脚本可保留为未挂载调试组件。
