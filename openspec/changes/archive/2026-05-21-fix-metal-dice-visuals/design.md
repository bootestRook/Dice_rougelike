## Context

当前青铜、黄金、水晶骰子使用材质流水线生成的 mesh、贴图、材质和预览场景；铁质骰子仍走 `GmDiceCtrl.gd` 内的简单 `StandardMaterial3D` 兜底路径。视觉验收显示，铁质稳定，而青铜和黄金在暗场或旋转角度下出现阴沉、亮暗跳变或主视面颜色混乱。

已定位的关键现状：

- 黄金 ORM 贴图几乎全图金属度为 1.0，当前 GM 场景缺少可控反射环境，暗面会因高金属 PBR 无环境可反射而塌黑。
- 青铜同时受 PBR 暗面问题和 albedo 铜绿覆盖影响，主视面存在偏绿偏脏的大片区域。
- 黄金 normal 贴图导入未按 normal map，且黄金/水晶导入未统一开启 mipmap；这会放大运动、远近和斜视角采样时的亮暗不稳定。
- 现有 Debug 测试主要检查资源存在、节点存在和基础字段，不足以捕捉“结构 PASS 但观感 FAIL”的问题。

## Goals / Non-Goals

**Goals:**

- 让黄金骰子在亮场、中性光、暗场下都保持黄金材质可读，暗面不塌黑，亮面不过曝。
- 让青铜骰子保留古旧青铜和局部铜绿，但不让铜绿大面积污染主视面。
- 统一骰子材质导入规则，确保 normal、mipmap、ORM/height 数据贴图配置稳定。
- 让 GM 物理骰子视口和材质预览场景为金属 PBR 提供足够的漫反射与反射参考。
- 扩展自动验收，覆盖材质通道统计、导入设置和暗场可读性。

**Non-Goals:**

- 不引入外部美术、字体、图标包或版权不明资源。
- 不把代码生成的临时素材当作最终正式美术，只在现有确定性流水线内调整参数与验收。
- 不改变战斗规则、奖励池、骰面槽位、骰胚规则或玩家操作流程。
- 不重写 GM 物理骰子系统或替换 Godot 4.x 渲染管线。
- 不为青铜/黄金引入自定义 shader，除非后续验收证明 Standard/ORM 材质无法满足目标。

## Decisions

### 1. 先修 PBR/场景匹配，再调贴图内容

青铜和黄金都属于金属材质，暗面可读性首先取决于 PBR 材质是否有合理的反射环境和补光。实现时优先在 GM 视口和预览场景中增加可控的中性反射/环境光策略，并保持多色补光不过度染色。

替代方案是大幅提高 emission 或把金属度降到接近非金属。该方案能快速变亮，但会把黄金变成自发光塑料感，不能解释真实金属暗面，因此不作为主路径。

### 2. 黄金降低“全图满金属”风险，而不是取消金属感

黄金仍应是高金属材质，但 metallic 不应全图硬锁 1.0；边缘、磨损与面内区域可有小幅差异。roughness 应保持中高范围，避免亮面硬闪，同时暗面依靠环境反射保持暖金色。

替代方案是只调灯光，不改 ORM。该方案会让黄金在某些角度仍然出现大块黑面，且无法通过贴图统计验收。

### 3. 青铜铜绿必须成为局部材质特征

青铜应保留铜绿，但铜绿覆盖率和位置需要受控：优先出现在边缘、划痕、噪声门控区域，不应在主视面形成大面积绿块。颜色应更接近旧铜、暗铜、局部氧化，而不是绿色涂层。

替代方案是直接移除铜绿。该方案会损失青铜与铁/黄金的风格差异。

### 4. 导入设置纳入流水线验收

材质生成脚本和测试需要检查 `.import` 关键字段：normal map、mipmap、ORM/height 数据贴图、normal y 方向。资源本体正确但导入配置不一致时，测试必须失败。

替代方案是只依赖 Godot 自动检测。当前黄金 normal 和 mipmap 已证明自动状态不可靠。

### 5. 增加数值化视觉验收，但保留人工最终审美

自动测试应检查暗场亮度下限、亮暗对比上限、铜绿覆盖率、金属度占比、粗糙度范围、导入设置和预览资源存在。最终观感仍需要打开 Godot 预览或 GM 场景确认，因为截图与 headless 环境无法完全代表实时渲染。

替代方案是只保留人工验收。该方案会让回归难以及时发现。

## Risks / Trade-offs

- 金属暗面变亮后可能削弱戏剧性对比 → 使用暗场最低亮度和最大对比阈值控制，不把暗场调成平光。
- 降低黄金 metallic 差异可能让黄金不够贵重 → 保留高金属主导，同时通过粗糙度、暖色反射和边缘磨损维持金器感。
- 收敛青铜铜绿可能让青铜和铁质接近 → 用旧铜 albedo、局部氧化和更高 roughness 区分。
- Headless Godot 可能无法可靠截取真实 PBR → 继续使用结构/通道测试，并在图形环境中人工打开预览场景确认。
- 直接编辑 `.import` 可能被 Godot 重导入覆盖 → 优先让生成/导入流程稳定复现，并在测试中检查最终导入状态。

## Migration Plan

1. 调整生成脚本中的黄金/青铜 PBR 参数和贴图生成逻辑。
2. 统一导入设置并通过 Godot 重导入或资源构建脚本刷新材质资源。
3. 调整 GM 视口和预览场景的反射/环境/补光配置。
4. 更新或新增 Debug 测试，覆盖导入设置和视觉数值阈值。
5. 复跑材质生成、资源构建、材质流水线测试、GM 光源测试、GM 编辑测试和主场景启动检查。
6. 如观感不达标，回滚本次资源生成结果和场景光照改动，保留测试阈值用于下一轮调参。

## Open Questions

- 青铜铜绿覆盖率目标需要最终美术口径确认；建议先以“主视面不形成连续大片绿块”为验收标准。
- 是否需要为 GM 视口增加 `ReflectionProbe`，还是只使用 `WorldEnvironment`/Sky/radiance 即可满足稳定性？
