# 骰子构筑 Roguelite：阶段性设计总结

> 版本：v0.3 设计口径重构  
> 日期：2026-05-16  
> 用途：替换旧版 `dice_roguelite_discussion_summary.md`  
> 当前结论：**纯骰子构筑；没有血量；没有独立锁定；玩家只追求目标战力；骰面维度收束为“点数 / 面饰 / 印记”；整颗骰子的扩展维度为“面数 / 骰胚”，当前有哪些点数直接由 `DieState.faces[].pip` 表达。**

---

## 0. 一句话版本

本项目是一个以目标分为核心的骰子构筑 Roguelite：

> **玩家用固定出战骰组进行每场战斗。每手投出所有出战骰，通过“选择骰子”决定重投或结算，在有限手数内累计战力达到目标。战斗后通过铸骰件改造骰面的点数、面饰和印记，逐步把普通骰组构筑成爆分引擎。**

当前不采用血量、护甲、敌人攻击、防御回合等攻防系统。失败条件只有一个：

```text
限定手数内没有达到目标战力。
```

---

## 1. 本版相对旧版的关键修正

旧版总结中仍保留了“锁骰”“骰面材质”“符文槽”“等级槽”“血量 / 扣血”等旧口径。本版统一修正如下。

| 旧口径 | 新口径 | 原因 |
|---|---|---|
| 锁骰 | 删除独立锁定操作 | 当前实际操作是“选择骰子后重投”或“选择骰子后结算” |
| 骰面材质 | 改为“面饰” | 材质更像整颗骰子的底盘，不适合只属于单个面 |
| 材质 / 骰胚并存 | 统一为“骰胚” | 骰胚是整颗骰子的底盘槽位，材质只是骰胚的一类表现 |
| 符文槽 | 不作为常规骰面维度 | 符文容易和面饰、印记抢职责，增加理解成本 |
| 等级槽 | 不作为常规骰面维度 | 单纯加数值无聊，强化规则又复杂；改为低频服务或骰型升级 |
| 血量 / 扣血 | 第一版排除 | 玩家目标只应是达到目标战力，不引入生存压力 |
| 花色替代物 | 不强行添加 | 骰子没有天然花色，应使用骰子的天然维度：面数、骰胚、逐面点数、重投、未结算留场 |
| D12 / D20 | 暂不采用 | 面数过高会削弱单个面的存在感；后续最多优先考虑 D8 |

## 2. 当前核心设计原则

### 2.1 玩家只有一个战斗目标

```text
在限定手数内累计战力达到本场目标分。
```

不做：

```text
血量
护甲
治疗
扣血换倍率
敌人攻击
防御回合
```

### 2.2 骰面保持简单，复杂构筑放到全局层

单个骰面应该 1 秒能读懂：

```text
6 / 爆裂面饰 / 红印
```

含义：

```text
这是一个 6。
它提供爆发收益。
它结算时会额外触发一次。
```

复杂连锁应该来自：

```text
骰面点数
+ 面饰收益
+ 印记触发
+ 骰胚特性
+ 骰具被动
+ 骰型升级
+ 重投 / 结算选择
```

而不是把大量条件都塞进一个骰面。

### 2.3 不硬补“骰子花色”

扑克牌天然有花色，骰子没有。不要为了模仿《小丑牌》而强行添加：

```text
火 / 水 / 风 / 土
红 / 蓝 / 绿 / 黄
星 / 月 / 日 / 影
```

骰子的等价扩展空间应该来自：

```text
面数：D4 / D6 / D8
骰胚：标准、铁质、玻璃、偏心、空心、镜面
当前骰面：这颗骰子的 `faces[]` 里有哪些点数、面饰、印记
操作状态：被重投、被结算、未结算留场、少选、初投出现
```

## 3. 当前推荐的系统分层

```text
整局目标：目标战力

战斗操作：
选择骰子 → 重投所选
选择骰子 → 结算所选

整颗骰子：
面数 / 骰胚

单个骰面：
点数 / 面饰 / 印记

全局构筑：
骰具 / 骰型升级 / 铸骰件 / 铸骰坊 / 低频整骰奖励
```

## 4. 与《小丑牌》的映射关系

本项目不应简单复制《小丑牌》的花色结构，而应参考它的分层方式。

| 《小丑牌》层级 | 骰子系统对应 | 说明 |
|---|---|---|
| Blind 分数门槛 | 本场目标战力 | 限定机会内打够分 |
| Hands | 每场出手次数 | 例如每场 4 手 |
| Discards | 重投次数 | 选择骰子后重投所选 |
| 选择牌出牌 | 选择骰子结算 | 核心计分动作 |
| 留在手牌中的牌 | 未结算留场骰面 | 投出但没有被结算的骰面，可触发留场收益 |
| Rank | 点数 | 1 / 2 / 3 / 4 / 5 / 6，未来可扩到 8 |
| Suit | 不强行映射 | 骰子没有天然花色 |
| Enhancement | 面饰 | 单面收益改造，例如筹码、倍率、爆裂、幸运 |
| Seal | 印记 | 单面触发改造，例如红印、蓝印、紫印 |
| Edition | 暂缓 | 可后续作为稀有强化，不进入当前核心 |
| Joker | 骰具 | 低频全局被动，承担复杂构筑方向 |
| Planet | 骰型升级 | 升级一对、三同、顺子等主组合收益 |
| Tarot / Spectral | 铸骰件 / 铸骰坊服务 | 改点数、加面饰、加印记、复制、删除、净化 |
| Deck composition | 骰组 / 当前骰面 | 逐面改造每颗骰子的概率结构 |
| Deck / Voucher / 特殊底子 | 骰胚 / 整骰低频奖励 | 改变整颗骰子的底盘性质 |

一句话映射：

```text
点数 ≈ Rank
面饰 ≈ Enhancement
印记 ≈ Seal
骰具 ≈ Joker
骰型升级 ≈ Planet
铸骰件 / 铸骰坊 ≈ Tarot / Spectral / 改牌服务
骰胚 + 面数 + 当前骰面 = 骰子自己的核心构筑空间；当前骰面不是独立类型，只是 `faces[]` 数据
```

## 5. 战斗操作模型

当前没有独立“锁定”。正式操作模型为：

```text
投出所有出战骰
↓
选择一批骰子
↓              ↓
重投所选        结算所选
```

同一个“选择”动作有两种用途：

| 玩家操作 | 结果 |
|---|---|
| 选择骰子 + 重投 | 被选骰子重新掷出 |
| 选择骰子 + 结算 | 被选骰子进入本手计分 |
| 没有被结算的骰子 | 成为未结算留场骰面，可触发留场类效果 |

建议使用的操作状态：

| 状态 | 含义 | 构筑价值 |
|---|---|---|
| 被重投 | 本手曾被玩家选中并重投 | 重投流、紫印、重投骰具 |
| 被结算 | 最终被选择计分 | 主得分来源 |
| 未结算留场 | 投出了，但没有被选择结算 | 蓝印、留场面饰、未结算流 |
| 少选 | 本手只结算 1–3 颗 | 少选高爆发流 |
| 初投出现 | 第一次投骰就出现 | 初投稳定流 |
| 最后一手 | 本场最后一次出手 | 目标分压力下的爆发条件 |

不再使用的术语：

```text
锁定
未锁定
锁定流
```

## 6. 整颗骰子层：面数、骰胚、当前面位

### 6.1 面数

面数是骰子最天然的扩展维度，优先考虑：

```text
D4 / D6 / D8
```

| 面数 | 定位 | 构筑含义 |
|---|---|---|
| D4 | 稳定核心骰 | 面少，核心面更容易掷出，适合高价值面 |
| D6 | 标准骰 | 均衡、默认、最适合基础组合 |
| D8 | 容量骰 | 面多，可容纳更多功能面，但核心面命中率更低 |

建议：D8 初期不必立刻引入 7、8 点。可以先做“8 个面位，但点数仍主要为 1–6”，这样不会破坏现有骰型概率。

### 6.2 骰胚

骰胚是整颗骰子的底盘槽位。材质词汇可以作为骰胚表现，但不要再把“材质”作为独立系统名。

```text
骰胚：标准 / 铁质 / 玻璃 / 偏心 / 空心 / 镜面
```

示例方向：

| 骰胚 | 直觉 | 可能方向 |
|---|---|---|
| 标准骰胚 | 无特殊底盘 | 默认、稳定 |
| 铁质骰胚 | 厚重、稳定 | 未结算留场收益、稳定倍率 |
| 玻璃骰胚 | 脆、高爆发 | 终倍率、破碎风险 |
| 偏心骰胚 | 概率被改变 | 某些面权重提高 |
| 空心骰胚 | 轻、适合重投 | 被重投后收益提高 |
| 镜面骰胚 | 对称、复制 | 面之间共享或映射效果 |

### 6.3 当前骰面

当前骰面是“这颗骰子里有哪些面位”。它不是独立类型，也没有额外的分布 ID；实际构成直接来自 `DieState.faces[].pip`。

示例：

```text
当前骰面：1 / 2 / 3 / 4 / 5 / 6
当前骰面：1 / 2 / 3 / 4 / 6 / 6
当前骰面：1 / 1 / 2 / 2 / 3 / 4
当前骰面：2 / 2 / 4 / 4 / 6 / 6
当前骰面：1 / 2 / 3 / 4 / 5 / 5
```

铸骰件改造的主要对象是具体面位：点数片直接改 `faces[i].pip`，面饰片和印记片直接改对应槽位。

## 7. 单个骰面层：点数、面饰、印记

### 7.1 点数

点数是骰面的基础身份。

```text
1 / 2 / 3 / 4 / 5 / 6
```

作用：

```text
形成一对、两对、三同、顺子、四同、五同
影响点数总和
进入高点、低点、奇数、偶数、六点等构筑轴
```

点数本身不要携带复杂效果。

### 7.2 面饰

面饰是单个骰面的收益改造，对应《小丑牌》的 Enhancement。

面饰回答：

```text
这个面在触发时提供什么收益？
```

建议方向：

| 面饰 | 效果方向 | 说明 |
|---|---|---|
| 无面饰 | 无额外收益 | 默认面 |
| 筹码面饰 | 计分时 +基础战力 | 稳定底分 |
| 倍率面饰 | 计分时 +倍率 | 稳定放大 |
| 爆裂面饰 | 计分时提供终倍率 | 高爆发，可考虑破碎风险 |
| 幸运面饰 | 概率给基础战力或倍率 | 随机高收益 |
| 留场面饰 | 未结算留场时提供收益 | 适配投 6 选 5 的不选决策 |
| 石质面饰 | 点数弱化但给大量基础战力 | 牺牲骰型换稳定分 |

命名注意：不要叫“骰面材质”。“材质 / 铁质 / 玻璃”属于整颗骰子的骰胚语义。单个面上的收益改造统一叫“面饰”。

### 7.3 印记

印记是单个骰面的触发方式改造，对应《小丑牌》的 Seal。

印记回答：

```text
这个面什么时候触发？是否额外触发？
```

建议方向：

| 印记 | 效果方向 | 说明 |
|---|---|---|
| 无印记 | 正常触发 | 默认 |
| 红印 | 被结算时额外触发一次 | 重触发核心 |
| 蓝印 | 未结算留场时触发弱收益 | 未结算流 |
| 紫印 | 被重投后出现时触发收益 | 重投流 |
| 金印 | 触发时给资源 | 经济系统实装后再考虑 |
| 白印 | 不受某些禁用影响 | Boss 对抗，后期再做 |

## 8. 不作为当前主维度的内容

### 8.1 符文

符文不作为常规骰面槽位。

原因：符文很容易和面饰、印记抢职责。

```text
六点符文 +倍率       → 更像面饰或骰具
对子符文额外触发     → 更像印记
顺子符文 +基础战力    → 更像面饰或骰型升级
```

后续如果保留“符文”一词，建议作为稀有面饰、特殊印记或骰具主题名，而不是第四个基础槽位。

### 8.2 等级

等级不作为常规骰面槽位。

原因：

```text
单纯 Lv2 +5 基础战力：无聊
强化具体规则：理解成本高
```

更合理的承载位置：

```text
骰型升级
铸骰坊低频强化服务
Boss 后系统奖励
稀有骰具
```

### 8.3 血量

第一版不引入血量。

风险收益可以通过以下方式实现：

```text
概率触发
破碎风险
机会成本
占用槽位
重投成本
目标分压力
```

不通过扣血实现。

## 9. 算分系统

### 9.1 核心公式

```text
最终战力 = Chips × Mult × XMult
```

中文显示建议：

| 内部名 | 中文名 | 说明 |
|---|---|---|
| Chips | 基础战力 | 底分 |
| Mult | 倍率 | 加算倍率 |
| XMult | 终倍率 | 乘算倍率，负责爆分 |
| Retrigger | 额外触发 | 重复触发某个面或效果 |
| Combo | 骰型 | 本手主组合 |

### 9.2 推荐结算顺序

```text
1. 读取本手被结算骰子
2. 判断主骰型
3. 记录条件标签
4. 建立基础 Chips / Mult / XMult
5. 计算被结算骰面的点数总和
6. 触发被结算骰面的面饰
7. 触发被结算骰面的印记
8. 触发未结算留场骰面的相关效果
9. 触发整颗骰子的骰胚效果
10. 触发骰具 / 角色 / 全局被动
11. 处理额外触发
12. 连乘 XMult
13. 写入结算日志
```

结算日志必须保留。玩家需要看懂：

```text
我为什么得了这么多分？
哪个骰面触发了？
哪个印记额外触发了？
哪个骰具给了倍率？
XMult 是怎么来的？
```

## 10. 主骰型与条件标签

### 10.1 主骰型只取一个

推荐优先级：

```text
五同
大顺
四同
葫芦
小顺
三同
两对
一对
高点
```

主骰型负责基础 Chips 和基础 Mult。

### 10.2 条件标签可以多个

条件标签不和主骰型抢身份。

例如：

```text
2、2、4、4、6
```

主骰型：

```text
两对
```

条件标签：

```text
全偶
含 6
高点
```

### 10.3 条件标签草案

| 标签 | 条件 | 作用 |
|---|---|---|
| 全奇 | 被结算骰全为奇数 | 奇数流 |
| 全偶 | 被结算骰全为偶数 | 偶数流 |
| 低点 | 被结算点数总和 ≤10 | 低点流 |
| 高点 | 被结算点数总和 ≥25 | 高点流 |
| 含 6 | 至少一个 6 | 六点流 |
| 多个 6 | 至少三个 6 | 六点爆发 |
| 少选 | 只结算 1–3 颗骰子 | 少选流 |
| 使用过重投 | 本手发生过重投 | 重投流 |
| 最后一手 | 本场最后一次出手 | 目标压力下的爆发条件 |
| 未结算留场 | 有投出但未结算的特殊面 | 留场流 |

## 11. 奖励与构筑件

### 11.1 普通战奖励

普通战后仍建议：

```text
三选一铸骰件
```

但铸骰件类型应更新为：

| 类型 | 作用 | 适合作为普通奖励 |
|---|---|---|
| 点数片 | 替换一个骰面的点数 | 是 |
| 面饰片 | 给一个骰面安装 / 替换面饰 | 是 |
| 印记片 | 给一个骰面安装 / 替换印记 | 是 |
| 复合件 | 同时改点数 + 面饰 / 印记 | 是，但偏稀有 |
| 复制件 | 复制一个骰面到另一个位置 | 偏稀有 |
| 净化件 | 移除负面面饰 / 负面印记 | 有负面系统后再增强 |
| 删除 / 重置件 | 删除或重置低价值面 | 更适合铸骰坊 |
| 骰胚更换 | 更换整颗骰子底盘 | 低频，不作为普通战固定奖励 |
| 面数改变 | D4 / D6 / D8 转换 | 低频系统奖励 |

不再作为普通骰面奖励：

```text
材质片
符文片
升级件
```

它们应迁移为：

```text
材质片 → 骰胚更换或面饰片
符文片 → 面饰 / 印记 / 骰具
升级件 → 骰型升级或铸骰坊低频服务
```

### 11.2 安装规则

单个骰面只有三个常规槽位：

```text
点数槽
面饰槽
印记槽
```

安装规则：

| 安装内容 | 行为 |
|---|---|
| 点数片 | 替换点数，不影响面饰和印记 |
| 面饰片 | 替换面饰，不影响点数和印记 |
| 印记片 | 替换印记，不影响点数和面饰 |
| 复合件 | 只修改声明的槽位，不清除未声明槽位 |
| 复制件 | 复制指定面的一部分或全部槽位，规则需明确 |
| 净化件 | 只清除负面槽位，不改变正常槽位 |

示例：

```text
当前面：
点数：1
面饰：爆裂
印记：无

安装：红印 6

结果：
点数：6
面饰：爆裂
印记：红印
```

## 12. 整颗骰子的低频奖励定位

整颗骰子不作为普通战高频奖励，但仍是重要的低频构筑对象。

来源：

```text
精英战
Boss 战
事件
宝箱
铸骰坊
特殊节点
```

低频奖励可以提供：

```text
新面数：D4 / D8
新骰胚：玻璃、铁质、偏心、空心、镜面
事件式批量修改点数
整骰重铸
替换出战骰
```

整颗骰子适合承担“底盘变化”，不适合每场普通战大量发放。

---

## 13. 骰具定位

骰具对应《小丑牌》的 Joker，是全局构筑核心。

骰具应该承担复杂一点的规则，而不是让单个骰面变复杂。

示例方向：

| 骰具 | 效果方向 | 流派 |
|---|---|---|
| 六面神像 | 每个被结算的 6 提供倍率 | 六点流 |
| 低点护符 | 总点数 ≤10 时提供终倍率 | 低点流 |
| 对子复读机 | 每组对子让一个骰面额外触发 | 同点流 |
| 顺子剧本 | 连续打出顺子时逐步增强 | 顺子流 |
| 重投骰杯 | 本手每次重投提供收益 | 重投流 |
| 留场齿轮 | 未结算留场面越多，倍率越高 | 留场流 |
| 少选王冠 | 只结算 1–3 颗时提供高收益 | 少选流 |

原则：

```text
骰面简单，骰具复杂。
骰面是组件，骰具定义流派。
```

---

## 14. 骰型升级定位

骰型升级对应《小丑牌》的 Planet。

它不需要占用骰面槽位，而是作为长期成长：

```text
一对等级 +1
三同等级 +1
小顺等级 +1
大顺等级 +1
全偶标签强化
低点标签强化
少选标签强化
```

它的作用是强化玩家已经形成的路线，而不是给每个骰面加一个“等级”。

## 15. 可形成的主要流派

### 15.1 六点流

核心目标：提高 6 的出现率，让 6 被多次结算和放大。

构筑件来源：

```text
点数片：6
红印 6
六点相关骰具
复制或替换具体面位为 6
倍率面饰 / 爆裂面饰
```

### 15.2 低点流

核心目标：低点不再只是低分，而是触发高倍率或高效率。

```text
点数片：1 / 2
低点相关骰具
少选收益
爆裂面饰
逐面替换为低域点数
```

### 15.3 同点流

核心目标：对子、三同、四同、五同。

```text
复制点数
双点分布
对子 / 三同骰具
红印重触发
骰型升级
```

### 15.4 顺子流

核心目标：稳定拼出连续点数。

```text
点数片：2 / 3 / 4 / 5
顺子相关骰具
顺子相关骰型升级
D8 容量骰补点
重投流支持
```

### 15.5 奇偶流

核心目标：全奇或全偶触发全局效果。

```text
奇数 / 偶数点数片
逐面替换为奇数或偶数点数
奇偶骰具
条件标签强化
```

### 15.6 重投流

核心目标：通过重投追求组合，并让重投本身产生收益。

```text
紫印
重投骰具
空心骰胚
被重投后出现的面饰收益
```

### 15.7 未结算留场流

核心目标：利用投出但不结算的骰面。

```text
蓝印
留场面饰
铁质骰胚
投 6 选 5 的不选决策
```

### 15.8 少选流

核心目标：只结算少数核心面，换取高倍率。

```text
红印核心面
爆裂面饰
少选骰具
D4 稳定核心骰
```

## 16. 当前建议的数据结构口径

### 16.1 DieState：整颗骰子

```text
face_count: int                # 4 / 6 / 8
body_id: StringName            # 骰胚，例如 standard / iron / glass / biased / hollow / mirror
faces: Array[FaceState]        # 骰面列表
```

### 16.2 FaceState：单个骰面

```text

```

### 16.3 旧字段迁移建议

| 旧字段 | 新去向 |
|---|---|
| material_id | 如果是整骰底盘，迁移到 DieState.body_id；如果是单面收益，迁移到 ornament_id |
| rune_id | 迁移到 ornament_id / mark_id / dice tool，视功能决定 |
| level | 不作为常规骰面字段；迁移到骰型升级或铸骰坊低频强化 |
| locked | 不作为战斗核心状态；操作改为选择后重投 / 选择后结算 |

## 19. 最终设计定位

> **这是一个以目标战力为唯一战斗目标的纯骰子构筑 Roguelite。玩家通过选择骰子进行重投或结算，在有限手数内达成目标分。骰面只保留点数、面饰、印记三个直观维度；整颗骰子通过面数、骰胚和当前面位形成骰子自己的构筑空间。普通战奖励为三选一铸骰件，用于改造现有骰面；复杂流派和爆分连锁主要由骰具、骰型升级和整骰底盘承载。**
