# Dice Material Pipeline

Use this skill for this repository's dice material production workflow. Treat `MATERIAL_PIPELINE.md` as the source of truth; this skill is the operational checklist for repeating or extending it.

## Workflow

1. Inspect the current material resources before changing anything:
   - `assets/textures/dice/{bronze,gold,crystal}`
   - `assets/materials/dice`
   - `assets/shaders/dice`
   - `assets/models/dice`
   - `assets/scenes/preview`
   - `scripts/ui/debug/gm_dice_port` when connecting materials to GM replacement.

2. Generate deterministic texture atlases and preview shot images:

```powershell
python tools\scene_builders\GenerateDiceMaterialTextures.py
```

3. Generate or refresh Godot resources:

```powershell
& (Get-Command godot).Source --headless --path . -s "tools\scene_builders\BuildDiceMaterialPipeline.gd"
```

If `godot` is unavailable, try `$env:GODOT_BIN` as required by the repo instructions and report the exact blocker.

4. Validate the base pipeline:

```powershell
& (Get-Command godot).Source --headless --path . -s "tests_or_debug/DebugDiceMaterialPipelineSmokeTest.gd"
& (Get-Command godot).Source --headless --path . --quit-after 3
```

5. When integrating with GM dice replacement, also validate:

```powershell
& (Get-Command godot).Source --headless --path . -s "tests_or_debug/DebugGmDiceEditSmokeTest.gd"
```

## Integration Points

- Material IDs and Chinese names live in `scripts/ui/debug/gm_dice_port/GmDiceDefinition.gd`.
- Dice body mesh/material assignment lives in `scripts/ui/debug/gm_dice_port/GmDiceCtrl.gd`.
- GM screen automation lives in `scripts/ui/debug/GmPhysicsDiceTestScreen.gd`.
- GM edit panel controls live in `scripts/ui/debug/gm_dice_port/GmDiceHud.gd`.
- Preview scenes and resources should be generated by the builder scripts, not by hand-writing large `.tscn` files.

## Resource Rules

- Keep material templates at `assets/materials/dice/{bronze_dice,gold_dice,crystal_dice}.tres`.
- Keep `crystal_dice.gdshader` under `assets/shaders/dice`.
- Keep texture names as `{material}_dice_{albedo,normal,orm,emission,height,flow_mask}.png`.
- Use ORM channel order: red = occlusion, green = roughness, blue = metallic.
- Put preview shot images under `assets/scenes/preview/preview_shots`; avoid `screenshots/` because this repo ignores that directory name.
- Headless Godot may use the dummy renderer, so do not depend on viewport capture for screenshot generation. Use the Python texture generator's preview outputs or run visual checks in the editor when needed.

## Guardrails

- Bronze and gold should stay on `ORMMaterial3D` or `StandardMaterial3D` unless the user asks for a shader variant.
- Crystal should stay on `ShaderMaterial` and expose `alpha_base`, `emission_power`, `fresnel_power`, `flow_speed`, `glow_color`, and `tint_color`.
- All player-visible labels added for GM/debug UI must be Chinese.
- Do not introduce external art, fonts, or icons without clear license attribution.
- Do not use ad hoc generated artwork as final art outside this deterministic pipeline unless the user explicitly asks.
